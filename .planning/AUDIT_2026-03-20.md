## 1. Sécurité & Privacy

### ✅ HIBP k-anonymity — CONFORME
- Seul le préfixe SHA-1 de 5 caractères est envoyé (ligne 1973 app.js)
- Header `Add-Padding: true` ajouté pour confidentialité additionnelle
- Aucun mot de passe complet transmis
- Race conditions prévenues avec `if (input.value !== pw) return`

### 🔴 CRITIQUE — CSP contient `'unsafe-eval'` (confiance: 95%)
**Fichier**: index.html ligne 7
**Problème**: La politique CSP définit `'unsafe-eval'` pour supporter TensorFlow.js, ce qui casse la protection XSS promise par CLAUDE.md ("Forbidden: `eval()`").

```html
<!-- AVANT (sécurisé) -->
script-src 'self' 'sha256-...' ...

<!-- APRÈS (compromis) -->
script-src 'self' 'unsafe-eval' 'sha256-...' ...
```

**Conséquence**: Un attaquant exploitant une vulnérabilité DOM injection pourrait utiliser `eval()`.

**Options de fix**:
1. Supprimer TensorFlow.js (restaure CSP sécurisée)
2. Utiliser TF.js en mode CPU-only sans eval (performances dégradées)
3. Documenter explicitement ce trade-off dans CLAUDE.md

### 🟡 Risque faible — Concaténation innerHTML sans échappement (confiance: 70%)
**Fichier**: app.js lignes 3578-3588, 3636-3658
**Détail**: Les labels de vulnérabilités sont insérés via `innerHTML` sans échappement. En pratique, les valeurs proviennent de traductions hardcodées dans le code source, donc pas exploitable. Mais le pattern est fragile par principe.

## 2. Qualité du Code

### 🔴 CRITIQUE — Crash sur mots de passe avec dates (confiance: 100%)
**Fichier**: app.js ligne 2847 & 2727
**Symptôme**: `TypeError: Cannot read properties of null` sur `null.match(...)`

```javascript
// BUGUÉ (ligne 2847)
const dateInfo = dt ? detectDateAndReduce(null, len, cs) : { hasDate: false, reduction: 1.0 };

// Dans detectDateAndReduce (ligne 2727)
const yearMatch = pw.match(/(?:19|20)\d{2}/); // pw est null → crash
```

**Test reproduction**: Entrer `Password2024!` → le calcul s'arrête avec une erreur non visible.

**Fix**:
```javascript
// Option 1: passer pw au lieu de null
const dateInfo = dt ? detectDateAndReduce(pw, len, cs) : { hasDate: false, reduction: 1.0 };

// Option 2: refactoriser detectDateAndReduce pour accepter un flag au lieu du mot de passe
```

### 🔴 CRITIQUE — Variable CSS `--radius-md` non définie (confiance: 100%)
**Fichier**: styles.css ligne 766
**Problème**:
```css
:root {
  --radius: 12px;
  --radius-sm: 8px;
  /* --radius-md: 10px;  ← MANQUANT */
}

.character-analysis-wrapper {
  border-radius: var(--radius-md); /* Se résout à 0px (valeur initiale) */
}
```

**Résultat**: Le wrapper "character analysis" a les coins carrés au lieu d'arrondis, cassant la cohérence visuelle.

**Fix**: Ajouter `--radius-md: 10px;` dans `:root`.

### 🟡 IMPORTANT — Shuffle du générateur biaisé (confiance: 85%)
**Fichier**: app.js lignes 3891-3895
**Problème**: Utiliser `.sort()` avec un comparateur aléatoire ne produit pas un shuffle uniforme (l'algo de tri suppose un ordre total cohérent).

```javascript
// BIAISÉ
const shuffled = password.split('').sort(() => {
    const arr = new Uint8Array(1);
    crypto.getRandomValues(arr);
    return arr[0] - 128;
}).join('');
```

**Raison supplémentaire**: `crypto.getRandomValues()` est appelée ~O(n log n) fois (à chaque comparaison du tri), ce qui viole la garantie que le comparateur `.sort()` doit être déterministe.

**Fix**: Utiliser Fisher-Yates avec appels de `crypto.getRandomValues` bien définis.

### 🟡 IMPORTANT — Variable morte `CHARSET_UNICODE_ESTIMATE` (confiance: 85%)
**Fichier**: app.js ligne 2068
**Détail**: Déclarée mais jamais utilisée. Seul `CHARSET_UNICODE` est référencé.

**Fix**: Supprimer la ligne 2068.

- [ ] Supprimer TensorFlow.js si la feature est expérimentale ou
- [ ] Documenter clairement dans CLAUDE.md que ML est désactivé

### 🟡 IMPORTANT — Biaises linguistiques dans `fmtDate` (confiance: 80%)
**Fichier**: app.js ligne 3215

```javascript
// Traite seulement FR et EN
d.toLocaleDateString(LANG === "fr" ? "fr-FR" : "en-US", {...});
```

Un utilisateur en polonais (`LANG === "pl"`) verra des dates au format américain au lieu du format polonais.

**Fix**:
```javascript
const localeMap = {
  en: "en-US", fr: "fr-FR", es: "es-ES", pt: "pt-BR",
  de: "de-DE", tr: "tr-TR", it: "it-IT", pl: "pl-PL", nl: "nl-NL"
};
d.toLocaleDateString(localeMap[LANG] || "en-US", {...});
```

## 5. Performance

### ⚠️ TensorFlow.js — Impact non-trivial (confiance: 95%)
**Détail**: Ajoute 3 MB + ~100-300 MB WebGL memory, ce qui dépasse le budget CLAUDE.md (<50 MB).

**Chargement**:

- Mais tape le réseau externe (jsdelivr) sur le premier mot de passe entré

**Recommandation**: Documenter ce trade-off ou considérer un chargement optionnel/feature flag.
