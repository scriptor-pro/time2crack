## Observation ouverte : word stretching

Test du mot de passe `leswaaaar.Le84` :
- Tous les modèles retournent `null` sauf brute force → rang 2.1×10²⁷ → ~14M années
- Cause : `leswaaaar` non détecté comme dictWord car les répétitions `aaaar` cassent la détection
- Le mot de base est probablement `les waaaar` / `le swaar` / variante de `swear`
- Piste : détecter les mots avec "character stretching" (répétitions de lettres) avant lookup dict
  - Algorithme : réduire les runs de 3+ caractères identiques à 1 ou 2 avant de chercher dans le dict
  - Ex : `waaaar` → `waar` → `war` → chercher `war` dans le dict
  - Référence : zxcvbn le fait nativement
