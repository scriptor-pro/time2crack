❌ BEFORE (Current UI)

Time2Crack v0.9.33 — Current production version with identified UX issues

🚨 Issues Identified

  • Attack table shows 60 cells immediately Hick's Law Miller's Law
  • App description breaks visual flow Law of Proximity
  • Info icon too small (14px) Fitts's Law
  • No explanations for jargon (PCFG, NTLM, Argon2id) Cognitive Load
  • Security warnings not visually grouped Law of Common Region
  • 3 competing primary actions Hick's Law
  • Equal visual weight on both result cards Von Restorff Effect
  • No progressive disclosure Cognitive Load

✅ AFTER (Improved UI)

Enhanced with Laws of UX principles — Reduced cognitive load, improved accessibility

🎯 Improvements Applied

  • Progressive disclosure: 12 cells initially, full 60-cell table opt-in (-80% cognitive load)
  • App description moved to header — Better visual flow between input and results
  • Info icon enlarged to 24px — WCAG 2.5.5 compliant touch target
  • Inline glossary tooltips — Hover on PCFG, NTLM, Argon2id shows explanation
  • Security warnings grouped — Bordered container with clear title
  • 2 primary actions (Test emphasized) — Reduced decision paralysis
  • Emphasized "Fastest Crack" card — Pulsing border, larger size, ⚡ icon
  • 3-level progressive disclosure: Summary → Top 3 Threats → Full Analysis
  • Strength icons added: ❌ Weak, ⚠️ Fair, ✓ Good, ✓✓ Strong, ⭐ Excellent
  • Fixed cyan contrast: #00d4ff → #00b8e6 (WCAG AAA compliant)
