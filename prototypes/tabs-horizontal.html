🔨 Brute Force
Teste toutes les combinaisons possibles de caractères : 'aaa', 'aab', 'aac'... C'est la méthode la plus exhaustive mais aussi la plus lente. Chaque caractère supplémentaire multiplie le temps par 94×. C'est pourquoi les mots de passe longs (12+ caractères) résistent des millions d'années même contre cette attaque.
📖 Dictionary
Utilise une base de 14+ milliards de mots de passe compromis issus de fuites de données (Have I Been Pwned). Teste d'abord les mots de passe les plus courants : 'password', '123456', 'qwerty'. Cette attaque est instantanée car elle cible les patterns humains prévisibles.
🔀 Hybrid
Combine attaque par dictionnaire et mutations. Prend un mot de base ('password') et applique ~1,000 variations : 'Password', 'password1', 'P@ssw0rd', 'password123'. Très efficace car les humains suivent des patterns prévisibles pour "renforcer" leurs mots de passe.
🎭 Mask
Exploite les structures prévisibles : Majuscule + minuscules + chiffres (ex: 'Password123'). Au lieu de tester toutes les combinaisons, l'attaque se concentre sur les patterns humains typiques, réduisant l'espace de recherche de 99%.
🌈 Rainbow Table
Tables précalculées de hashes pour les mots de passe courts (≤8 caractères). Permet une recherche instantanée : au lieu de calculer le hash, on le cherche dans une énorme base de données. Inefficace contre les algorithmes modernes avec sel (bcrypt, Argon2id).
